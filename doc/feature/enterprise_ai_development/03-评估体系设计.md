## 项目背景

**项目名称**：pet-care-ai
**技术栈**：Spring AI + 智谱AI + Elasticsearch + Redis
**当前架构**：单步 Function Calling，无量化评估
**本文档目标**：建立 RAG 和工具调用的量化评估体系

---

## 一、现状分析

### 1.1 当前问题

| 问题 | 描述 | 影响 |
|------|------|------|
| 无量化指标 | 不知道 RAG 检索准不准 | 无法优化 |
| 无质量评估 | 不知道回答质量如何 | 无法发现问题 |
| 无工具评估 | 不知道工具调用对不对 | 调试困难 |
| 依赖人工判断 | 只能靠感觉 | 无法客观对比 |

### 1.2 企业级需要监控的指标

| 维度 | 指标 | 说明 |
|------|------|------|
| **检索质量** | Recall@K、Precision@K、MRR | RAG 召回质量 |
| **RAG 忠实度** | Faithfulness、Answer Relevance | 回答是否基于检索内容 |
| **工具调用** | 成功率、参数错误率、误调用率 | Tool Calling 准确性 |
| **用户满意度** | 点赞/点踩、会话重开率 | 用户体验 |

## 三、RAG 评估

### 3.1 RAGAS 指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **Recall@K** | 召回率，金标准文档被召回的比例 | (召回的文档数) / (金标准文档数) |
| **Precision@K** | 精度，召回文档中相关的比例 | (相关文档数) / (召回文档数) |
| **MRR** | 平均倒数排名，第一个相关文档的平均排名 | 1 / (第一个相关文档位置) |
| **Faithfulness** | 忠实度，回答是否基于检索内容 | LLM 评估生成内容的依据 |
| **Answer Relevance** | 回答相关性，回答与问题的相关度 | 向量相似度 |

### 3.2 评估数据集

**金标准构建**：
```
问题 + 相关文档列表（人工标注）+ 期望答案要点
```

**数据来源**：
- 用户高频问题（从 chat_trace 聚合）
- 人工标注（运营/客服团队）
- 合成数据（LLM 生成 + 人工审核）

### 3.3 评估流程

```java
// 伪代码
for (EvaluationCase case : dataset) {
    // 1. 执行检索
    List<Document> retrieved = vectorStore.similaritySearch(case.query());

    // 2. 计算 Recall@K / Precision@K
    Metrics metrics = calculateRetrievalMetrics(retrieved, case.relevantDocs());

    // 3. 执行 RAG
    String answer = chatClient.prompt().user(case.query()).call().content();

    // 4. 计算 Faithfulness / Answer Relevance
    Metrics ragMetrics = ragas.evaluate(case.query(), retrieved, answer);

    // 5. 保存结果
    saveEvaluationResult(case.traceId(), metrics);
}
```

## 四、工具调用评估

### 4.1 评估指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **成功率** | 工具调用成功的比例 | (成功次数) / (总调用次数) |
| **参数正确率** | 参数格式正确的比例 | (参数正确次数) / (总调用次数) |
| **误调用率** | 不该调用时调用的比例 | (误调用次数) / (总请求次数) |
| **调用延迟** | 工具平均响应时间 | (总耗时) / (调用次数) |
| **工具覆盖率** | 各工具被调用频率 | 各工具调用次数 / 总调用次数 |

### 4.2 金标准构建

**人工标注**：
```
问题 → 应该调用哪个工具 → 预期参数
```

**示例**：
```
"周末北京有什么宠物活动？"
→ 应该调用: searchActivities
→ 参数: query="宠物活动", startTime=2026-03-01, endTime=2026-03-02
```

### 4.3 评估方式

**基于 chat_trace 数据**：
```java
// 伪代码
for (ChatTrace trace : sampledTraces) {
    for (ToolCall toolCall : trace.getToolCalls()) {
        // 1. 检查是否应该调用
        boolean shouldCall = goldStandard.shouldCallTool(trace.getRequest(), toolCall.getToolName());

        // 2. 检查参数是否正确
        boolean paramsCorrect = goldStandard.validateParams(trace.getRequest(), toolCall);

        // 3. 检查调用是否成功
        boolean success = toolCall.isSuccess();

        // 4. 记录评估结果
        recordToolEvaluation(trace.getTraceId(), toolCall, shouldCall, paramsCorrect, success);
    }
}
```

    // 2. 执行评估

    // 3. 保存结果

## 十、风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 金标准构建成本高 | 人力投入大 | 半自动标注 + 众包 |
| RAGAS 评估延迟 | 评估耗时长 | 异步执行，不影响线上 |
| 评估指标不准 | 误导优化 | 多指标综合 + 人工复核 |
| 数据偏差 | 金标准不代表真实场景 | 定期更新数据集 |
